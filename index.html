SNS 自動化 Claude API — Video 01

テーマ入力だけで投稿文を自動生成：SNS 投稿文ジェネレーター

Claude API / Vercel Serverless / HTML/CSS/JS / プロンプトエンジニアリング

▲ SNS 投稿文生成デモ：テーマ・ターゲット・トーンを入力するだけで、Threads・Instagram・X（旧Twitter）向けに最適化された投稿文が即時生成されるまでを実演しています。

Problem
SNS の継続投稿は「何を書くか」「各プラットフォームに合わせて書き直す」という二重の工数が発生しがちで、発信が続かない原因になっていた。ツールによる自動投稿はあっても、「質の高い文章を量産する」工程は依然として人手に依存していた。
Solution
Claude API を活用し、テーマ・ターゲット・トーン（ビジネス／カジュアルなど）を入力するだけで、Threads・Instagram・X それぞれの文字数制限・文体・ハッシュタグ慣習に最適化された投稿文を一括生成するブラウザ完結型ツールを設計・実装。Vercel Serverless Functions を API 層に置くことで、環境構築不要・URL だけで即使える体験を実現。
Result
「書く」作業が「テーマを決める」作業に変わり、投稿文の量産と継続発信が可能に。プラットフォームごとの書き直し工数をゼロにし、SNS 運用の継続率向上と発信の一貫性を同時に実現。個人事業主・小規模チームの情報発信コストを大幅に削減。

設計思想：「完璧な一本を書く」のではなく「複数プラットフォーム向けをまとめて量産する」というコンセプトが核心。各 SNS の特性（X の簡潔さ・Instagram のハッシュタグ文化・Threads の会話調）をプロンプトに組み込み、出力品質を均一化しました。発信の「継続」こそが最大の成果につながるという考えから、摩擦ゼロの UX を最優先に設計しています。

SNS 自動化 Claude API Notion — Video 02

AI人格を蓄積して投稿を量産：Sayaka Auto Ver3

Claude API / Notion API / Vercel / Next.js / プロンプトエンジニアリング / SNS 自動化

▲ Sayaka Auto Ver3 デモ：テーマ・時間帯・商品情報を入力するだけで、Threads・Instagram 向けの投稿文・ハッシュタグ・リール文・画像生成プロンプトまで一括生成される様子を実演しています。

Problem
SNS 発信では、「毎回ゼロから考える」こと自体が継続の大きな負担になっていた。さらに、投稿ごとに空気感・口調・世界観がブレやすく、"同じ人格として発信し続ける"ことが難しいという課題があった。特に Threads・Instagram・リール向けテキストでは、媒体ごとの最適化と世界観維持を両立する必要があり、発信者の負担が大きくなっていた。
Solution
Claude API を活用し、テーマ・時間帯・商品情報を入力するだけで、Threads 投稿文・Instagram 用本文・ハッシュタグ・リール短文・画像生成プロンプトを一括生成するブラウザ完結型ツールを設計・実装。さらに、Notion に過去投稿・人格設定・トーン・発信ルールを蓄積し、「Sayaka」という AI 人格を継続的に再利用できる構造を構築。単なる文章生成ではなく、"同じ人格が発信しているように見える一貫性"を重視したプロンプト設計を行った。Vercel 上へデプロイすることで、環境構築不要・URL だけで即利用できる運用も実現。
Result
「毎回考える」作業を大幅に削減し、テーマ入力だけで複数 SNS 向けコンテンツを継続量産できる状態を実現。投稿頻度の維持だけでなく、人格・世界観・言葉遣いの統一によってブランド一貫性も向上。また、Notion に投稿資産を蓄積することで、過去コンテンツを AI が再利用しながら継続的に"人格成長"できる基盤を構築。過去投稿との重複や表現パターンの偏りも参照しながら生成を行うことで、「同じことばかり言ってしまう」問題を軽減し、長期運用でも発信の鮮度を維持できる構造を実現した。個人発信者でも、少人数運用で長期的な SNS ブランド構築が可能になった。

設計思想：「完璧な一本を書く」のではなく、「人格と世界観を維持したまま継続量産する」ことをコンセプトに設計。Threads の会話感・Instagram の情緒性・リール向け短文テンポなど、各媒体ごとの特性をプロンプトへ組み込みながらも、"Sayakaらしさ"が崩れない構造を重視した。また、Notion を AI の記憶領域として活用することで、単発生成ではなく「蓄積→再利用→発信改善」が循環する AI 人格運用を実現。発信継続の摩擦を極限まで減らす UX を最優先に設計している。

Architecture

構成図・設計図

「課題の抽象化 → 解決策の設計」のプロセスを図解で整理しています。

Multi-ChannelMake— Arch 01

マルチチャネル・コンテンツ配信パイプライン

Make / LINE / Buffer / OpenAI / 画像生成 AI

▲ Make ワークフロー：LINE 入力から SNS 自動投稿・画像生成まで一気通貫で処理するパイプラインの全体像です。

Problem
各 SNS ごとの投稿工数とトンマナ管理が煩雑。プラットフォームごとに原稿を書き直す二重工数が発生していた。
Solution
LINE からテーマ入力 → 最新情報を自動検索 → X 投稿文・画像を AI 生成 → Router で Buffer / LINE 返信に振り分ける抽象化フローを設計。
Result
単一の操作で複数チャネルへ最適化されたコンテンツを配信。投稿継続率が向上し、ブランドの一貫性を維持。

設計思想：「単一コンテンツを各 SNS 向けに最適化する抽象化フロー」が設計の核心。ツールに依存しない構造で、将来のチャネル追加にも柔軟に対応できます。

Auto MinutesMake— Arch 02

音声ファイルから自動生成：完全自動化 議事録作成システム

Make / Dropbox / AssemblyAI / OpenAI / Slack

▲ Make ワークフロー：音声ファイルのアップロードをトリガーに、文字起こし→議事録生成→Slack通知まで完全自動で処理するパイプラインの全体像です。

Problem
会議後の議事録作成が担当者の手作業に依存。文字起こし・要約・共有に時間がかかり、意思決定のスピードを阻害していた。
Solution
指定フォルダへの音声ファイル（mp3/m4a）アップロードをトリガーに自動起動。AssemblyAI で高精度な文字起こしを行い、GPT-4 が「決定事項」「ToDo」「次回アジェンダ」などのビジネスフォーマットで要約。生成した議事録をドキュメント保存し、Slack へ完了通知を送信。
Result
「会議後の議事録作成」という非生産的な作業時間をゼロに。チームへの共有も即時完了し、意思決定スピードを飛躍的に向上。

設計思想：「ファイルを置くだけ」という最小の操作で全工程が完結するゼロタッチ設計を追求。AI の出力フォーマットもビジネス実務に即した構造に固定し、すぐに使える議事録品質を担保しました。
